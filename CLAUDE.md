# Instructions

- **No defensive wrapping.** Do not add try/except, try/catch, or any error-handling wrappers unless explicitly asked. If something fails, surface the raw error.
- **Ask for clarification.** If a prompt is ambiguous or unclear, ask before proceeding rather than guessing.
- **No unsolicited design suggestions.** Follow the given design direction exactly. Do not propose alternative designs, patterns, or improvements unless asked.
- **Deliver exactly what is requested.** Execute the prompt precisely as stated — no additions, no extras, no creative liberties.
- **Flag infeasibility early.** If an instruction cannot be done or has a hard blocker, say so immediately instead of attempting a workaround.
- **Keep code comments plain and minimal.** No decorative comments, no emojis anywhere — including in UI/UX output.
